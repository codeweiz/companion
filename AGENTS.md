# Companion Project

`companion` is a native Android application written in Kotlin, bootstrapped from Google's
[Now in Android](https://github.com/android/nowinandroid) sample and trimmed to keep the main
application, modular architecture, and developer-critical quality gates. See `README.md` for the
project scope and what was removed from the upstream sample.

## Architecture

A modern, reactive, single-activity app that follows Google's official architecture guidance:

-   **UI:** Jetpack Compose with Material 3 and adaptive layouts for different screen sizes.
-   **State Management:** Unidirectional Data Flow (UDF) with Kotlin Coroutines and `Flow`s.
    `ViewModel`s act as state holders, exposing UI state as streams of data.
-   **Dependency Injection:** Hilt throughout the app.
-   **Navigation:** Jetpack Navigation 3 for Compose.
-   **Data:** Repository pattern. Local data with Room and DataStore; remote data with Retrofit and
    OkHttp.
-   **Background Processing:** WorkManager for deferrable background tasks.

## Naming Conventions

-   Base package / namespace / applicationId: `com.microboat.companion`.
-   Shared design-system and app-level components use the `App` prefix (e.g. `AppTheme`,
    `AppButton`, `AppIcons`, `AppBackground`). The root composable is `App`, its state holder is
    `AppState`, and the `Application` subclass is `CompanionApplication`.
-   Gradle convention plugins use the `companion.*` plugin id namespace (e.g.
    `companion.android.application`), declared in `build-logic/`.

## Modules

The main Android app lives in `app/`. Feature modules live in `feature/` (split into `api`/`impl`)
and core/shared modules in `core/`. Build logic lives in `build-logic/`.

## Commands to Build & Test

Two product flavors (`demo`, `prod`) and two build types (`debug`, `release`). JDK 21 is required.

- Build: `./gradlew assemble{Variant}`. Typically `assembleDemoDebug`.
- Fix linting/formatting: `./gradlew spotlessApply`
- Run local tests: `./gradlew test{Variant}UnitTest`, e.g. `testDemoDebugUnitTest`
- Run a single test: `./gradlew testDemoDebugUnitTest --tests "com.microboat.companion.MyTestClass"`
- Record/verify local screenshot tests: `./gradlew recordRoborazziDemoDebug` /
  `./gradlew verifyRoborazziDemoDebug`
- Dependency baseline: `./gradlew dependencyGuard`

### Instrumented tests

- Instrumentation tests: `./gradlew connectedDemoDebugAndroidTest`.

### Creating tests

#### Instrumented tests

- Tests for UI features should only use `ComposeTestRule` with a `ComponentActivity`.
- Bigger tests live in the `:app` module and they can start activities like `MainActivity`.

#### Local tests

- [kotlinx.coroutines](https://github.com/Kotlin/kotlinx.coroutines) for most assertions
- [cashapp/turbine](https://github.com/cashapp/turbine) for complex coroutine tests
- [google/truth](https://github.com/google/truth) for assertions

## Continuous integration

- Workflows are defined in `.github/workflows/*` and contain format, dependency guard, lint, unit
  test, screenshot test, assemble, and instrumentation checks.
- Screenshot golden images are generated by CI; avoid committing workstation-generated screenshots
  unless you are intentionally rebaselining them.
